À propos de ce projet

Présentation Contributeurs Licence
Notre projet "Challenge Open Data" porte sur les résultats du premier tous des élections régionales de 2015. Le but de ce projet est de croiser des données micro et macro économiques telles que le taux de chômage, le reveu médian et le taux d'immigration dans l'optique de trouver certaines corrélations.

Source des données

  • Données sur le chômage
  • Données d’immigration par départements et régions
  • Données sur le revenu médian par départements et régions
  • Résultats du tour 1 pour les élections départementales 2015

Transformation des données bruts

Dans un premier temps nous avons du transformer les données que nous avons récupérés. En effet, celles-ci comportaient beaucoup d'informations sans réelles importances et se trouvaient dans différents formats (CSV, JSON, XLS).
Nous avons donc codé un parser en Java permettant de récupérer les données que nous pensons intéressantes dans chacun de nos fichiers. Le principale avantage de ce parser et qu'il permet de rajouter facilement des sources de données diverses.
Pour les données de votes, nous les avons récupérés par communes, puis aggrégé pour avoir des résultats par département et par région.
Pour les autres données, notamment les critères sur l'axe des ordonnées (taux de chomage, taux d'immigration et revenu médian) nous avons parfois dû appliquer quelques petites transformations, lorsque celle-ci n'était pas directement retrouvables dans nos sources. Par exemple, pour obtenir une approximation du revenu médian par région nous avons dû ponderer le revenu médian de chacun des départements de cette région.

Stockage

Le parser permet comme nous l'avons vu de transformer nos données bruts pour ensuite les stocker dans une base de données MongoDB comportant 4 tables :
  • Départements
  • Régions
  • Communes
  • Partis

Technologies utilisées

La base de données utilisée est une base MongoDB, avec importations de fichiers .csv et .json. Le parsage a été effectué en langage Java.
Au niveau du serveur, nous avons implementé une API RESTful utilisant l'environnement JavaScript Node.js et le framework Express. Ce serveur communique avec la base MongoDB via des modèles définis pour assurer la validité des données, et réceptionne les requêtes de l'application via des routes définies.
